CSS选择器

CSS选择器一共包括四类：ID选择器，标签选择器、类选择器以及通配符选择器

ID选择器

以#开头定义ID选择器，通过id=引用该样式，仅可使用一次；

标签选择器

以HTML标签名定义，多所有该标签名均有效；

类选择器

以.开头，名称支持用户自定义，通过class=引用该样式定义,支持复用；

通配符选择器

定义名称为*,针对所有标签样式生效，无需单独引用；

<!DOCTYPE html>

<html lang="en">

<head> <meta charset="UTF-8" />

<link rel="icon" type="image/svg+xml" href="/vite.svg" />

<meta name="viewport" content="width=device-width, initial-scale=1.0" />

<title>Vite + Vue + TS</title>

<style>

/* id选择器 */

#app {

color: red;

}

/* 标签选择器 */

div {

color: gray;

}

  

/* 类选择器 */

.demo {

color: gray;

}

/* 通配符选择器 */

* {

color: black;

}

  

</style>

</head> <body> <div id="app">id选择器，仅可调用一次</div>

<div>标签选择自动生效</div>

<div class="demo">类选择器</div>

<script type="module" src="/src/main.ts"></script>

</body></html>

CSS元素显示模式

元素显示模式主要分为块元素和行元素；

块元素

  1. 块元素比较霸道，自己独占一行；
  2. 块元素可自定义高度、宽度及内外边距大小；
  3. 块元素宽度默认与父级元素保持大小一致；
  4. 块元素是一个容器级盒子，里面可以放行级元素或块级元素；

    文字类标签内部不允许放块级元素，如<p>,<h>等；

行元素

  1. 相邻行元素在同一行上，一行可以显示多个行元素；
  2. 高、宽无法直接设置；
  3. 宽度为元素本身内容的宽度；
  4. 行元素内只能放文字或其他行元素；

    链接内不能再放链接

行块元素互转

div {

color: gray;
/* 块转行元素*/
display: inline;

}


p {
weight: 100px;
height: 100px;
color: gray;
/* 行转块元素 */
display: block;

}

span {
	weight: 100px;
	height: 100px;
	color: gray;
	/* 行转行内块元素 */
	display: inline-block;
}

CSS背景

{
	background-color: red;
	background-repeat: no-repeat;
	background-image: url('image/logo.png')
	/* 支持参数left right top bottom center，如省略参数，则默认省略方向为居中 */
	/* 支持精确单位指定，第一个参数为x轴坐标，第二个为y轴坐标（可省略），如省略代表垂直居中 */
	/* 参数与精确单位可混合使用 */
	background-position: top center;
	/* 定义背景图片是跟随滚动还是固定 */
	background-attachment: fixed;
}

以上5个属性可合并在一个属性中：background: 颜色 图片 是否重复 位置 是否固定

CSS三大特性

层叠性

相同选择器，冲突样式采用就近原则，不冲突则全部生效；与结构最近的样式将会生效；

继承性

子标签会继承父标签的颜色、字号等文字相关样式；
行高可继承，但父级需显示指定行高，行高不带单位，则为当前文字大小的对应倍数；

优先级

选择器相同，执行层叠性；
不同选择器，则按照权重执行；
| 选择器 | 权重 |
| —————- | ——- |
| 继承或通配选择器 | 0,0,0,0 |
| 元素选择器 | 0,0,0,1 |
| 类选择器 | 0,0,1,0 |
| ID选择器 | 0,1,0,0 |
| 行内样式 | 1,0,0,0 |
| !important | 无穷大 |

权重叠加

复合选择器由复合层级各自的权重进行叠加计算后进行优先级比较；但不存在进位；

CSS盒子模型

CSS盒子模型的本质就是一个盒子，包括内外边距，边框及内容；

边框-Border

边框由粗细、样式及颜色组成；

  • border-width：以像素为单位；
  • border-style：由枚举参数指定，常用参数有：无-none，实线-solid，虚线-dashed，点线-dotted；
  • border-color：支持枚举颜色及十六进制编码；
  • border-radius：圆角边框，参数为内切圆半径长度或百分比（CSS3特性）；
  1. border-radius设置为50%，则为圆形；
  2. border-radius设置为高度的一半，则为圆角矩形；
  3. border-radius后可接多个参数，与内外边距原理一致，可设置不同角的弧度；
/* 完整模式 */
.demo {
		border-width: 10px;
		border-style: solid;
		border-color: red;
		border-radius：10px;
}

/* 简写模式 */
.demo1 {
		border: 12px solid red;
		border-radius：10px;
}

/* 单独指定上边框 */
.demo2 {
		border-top: 1px solid greed;
}

边框宽度会影响盒子实际大小；

内边距-Padding

内边距是边框与内容之间的距离；

  • padding-left：距离左侧边框距离
  • padding-right：距离右侧边框距离
  • padding-top：距离上侧边框距离
  • padding-bottom：距离下侧边框距离
简写方式 表达意思
padding： 5px 表示上下左右内边距都为5px
padding： 5px 10px 表示上下内边距为5px，左右内边距为10px
padding： 5px 10px 20px 表示上内边距为5px 左右内边距为10px 下内边距为20px
padding：5px 10px 20px 30px 表示从上开始按顺时针方向对应内边距
  1. 内边距会影响盒子实际大小；
  2. 不指定盒子的width/height，盒子继承父级大小，而设置内边距后，向内坍缩，不影响盒子大小；

外边距-Margin

外边距是盒子与盒子之间的距离；

  • margin-left：距离左侧盒子距离
  • margin-right：距离右侧盒子距离
  • margin-top：距离上边盒子距离
  • margin-bottom：距离下边盒子距离
    简写方式与padding一致；

典型应用

盒子水平居中

通过外边距让块级盒子水平居中；

  • 盒子必须设置宽度；
  • 盒子左右外边距设置为auto
    .demo {
		width：100px；
		margin：0 auto;  /* 其中0 可以省略 */
}

行内元素水平居中

行内元素或行内块元素通过给其父元素添加text-align: center实现；

.demo {
		width：100px；
		margin：0 auto;  /* 其中0 可以省略 */
		text-align: center;
}

<div class='demo'> 
	<span>行内元素水平居中</span>
</div>

嵌套盒子

两个盒子存在嵌套关系时，父盒子与子盒子同时配置上外边距时，父元素会向下塌陷两个外边距较大值；
解决方案为：

  1. 对父盒子设置边框；
  2. 对父盒子设置内边距；
  3. 对父盒子添加overflow:hidden；—常用
  4. 设置浮动、固定等，绝对定位盒子不存在塌陷问题；

通用样式

  1. 因不同浏览器对HTML标签设置了默认的内外边距，为避免不同浏览器兼容问题，可通过通配符选择器设置所有页面内外边距为0，有需要设置边距，可以在对应样式中进行覆盖；
  2. li标签自动添加的圆点多数情况需去除；
* {  
		margin: 0; 
		padding: 0; 
	}
	
li {
		list-style: none;
}

盒子阴影

盒子支持配置阴影属性；
box-shadow：水平位置 垂直位置 模糊程度 阴影大小 阴影颜色 内外阴影

div: hover {
	 box-shadow: 10px 10px 10px 4px gray inset
}

CSS布局

CSS布局传统模式有三种：标准流、浮动、定位；

标准流

所谓标准流，就是HTML标签按照默认方式排列；

浮动

float属性用于创建浮动框，将其移动到一边，直到左边缘或右边缘触及包含块或另一个浮动框的边缘；
float：枚举值参数-none，left及right；

浮动特性

  1. 元素添加了浮动属性后，将脱离标准流的控制移动到指定的位置，俗称”脱标“；
  2. 浮动的盒子不在保留原先的位置；其原先占有的位置将被其他标准流进行补位，即浮动盒子与标准流将会出现层叠（浮动仅会与其后标准流出现层叠）；
  3. 如果多个盒子同时设置浮动，将按照盒子上沿对齐在一行内显示，但超出父级宽度，则另起一行显示；
  4. 浮动元素具备行内块属性；

典型应用

  1. 标准流父元素排列上下位置，内部元素采用浮动排列左右位置；
  2. 先设置盒子大小，再设置盒子位置；
  3. 同一层级盒子内所有元素需保持浮动一致开启或关闭；

清除浮动

当父级不便于指定高度时，需通过清除浮动，让父级根据浮动的子盒子自动检测高度，避免影响父级同级盒子位置错乱；
clear：参数为枚举值，[both、left、right]

清除浮动方法

  1. 额外标签法，也称为隔墙法；缺点为增加额外标签，结构化差；
  2. 父级添加overflow属性；缺点为无法显示溢出部分；
  3. 父级添加after伪元素；兼容性差；
  4. 父级添加双伪元素；兼容性差；
.parent {

width: 200px;

overflow: hidden;

}

/* 伪元素 */

.clearfix: after {

content: "",

display: block;

height: 0;

clear: both;

visibility: hidden;

}

/* 双伪元素 */

.clearfix:before, .clearfix:after {

content: "";

display: table;

}

clearfix {

*zoom: 1;

}

.child1 {

float: left;

}

.child2 {

float: left;

}

.child3 {

float: left;

}

.child4 {

float: left;

}

/* 隔墙法，必须为块元素 */

.clear {

clear: both;

}

<div class="parent clearfix">

		<div class="child1">child1</div>

		<div class="child2">child2</div>

		<div class="child3">child3</div>

		<div class="child4">child4</div>

		<div class="clear">clear</div>

</div>

定位

定位 = 定位模式 + 边偏移；
定位模式用于指定一个元素在文档中的定位方式；
边偏移则决定了该元素的最终位置；

静态定位

静态定位是元素的默认定位方式，无定位的意思；
静态定位按照标准流摆放位置，无边偏移；

相对定位

相对定位是元素在移动位置的时候，是相对于它原来的位置（未配置定位前的位置）来说的；
相对定位移动后，原来在标准流的位置继续占有，其他标准流仍然按照该位置存在元素对待；（不脱标）

绝对定位

绝对定位是元素在移动位置的时候，相对于它的父级元素来说的；
如果没有父级元素或父级元素没有定位，则以浏览器为标准进行偏移；
如果父级元素有定位，则以最近一级有定位的父级元素为标准进行偏移；
决定定位移动后，不再占有标准流位置；（脱标）

固定定位

固定定位是元素固定在浏览器可视区位置；主要使用场景为：在浏览器滚动时元素的位置不会变动；
以浏览器的可视窗口为标准进行偏移；
跟父元素没有关系；
不随滚动条变化；
不占有原先的位置；（脱标）

粘性定位

粘性定位可以被认为是相对定位与固定定位的混合体；
以浏览器的可视区为参考点移动；
占有标准流位置；（不脱标）
必须添加left、right、top、bottom中一个才会生效；

定位叠放

在使用定位布局时，可能出现盒子重叠现象，此时，可使用z-index来控制盒子的前后次序（z轴）

  • z-index取值为整数，包括正整数、0及负整数，默认为auto；数值越大，层级越高；
  • z-index数值相同，按照书写顺序，后来居上；

布局原则

上下或左右排列，使用标准流布局；
多个块级元素使用浮动布局；
元素自由在盒子内移动使用定位布局；

小技巧

子绝父相：子级元素使用决定定位，父级元素使用相对定位；

.parent {

position: relative;

top: 100px;

}

.child {

position: absolute;

right: -4px;

}

<div class="parent">

<div class="child"></div>

</div>

绝对定位水平居中：添加定位后，无法通过margin： auto实现水平居中；需通过算法实现；

.middle {
		position: absolute;
		left: 50%;   /* 左偏移父级宽度一半 */
		margin-left: -100px;  /* 自身宽度的一半 */  
		
}

浮动与标准流重叠：浮动元素会覆盖标准流的元素，但标准流中的文字或图片会自动偏移让出浮动元素长度，文字会图片会完整显示；浮动相当于文字环绕效果；

元素显示与隐藏

display

display设置元素如何显示，参数为枚举值；

  • none：隐藏（脱标）；
  • block：转换该元素为块级元素并显示；

visibility

visibility指定一个元素是可见还是隐藏，参数为枚举值；

  • visible：元素可见；
  • hidden：元素隐藏（不脱标）；

overflow

overflow指定一个元素溢出元素部分是否隐藏，参数为枚举值；

  • auto：存在溢出部分显示滚动条，没有溢出部分不显示滚动条；
  • hidden：不显示溢出部分；
  • scroll：总是显示滚动条，用于滚动查看溢出部分；

CSS属性书写顺序

  1. 布局定位属性：display、position、float、clear、visibility、overflow等，建议display第一个写；
  2. 自身属性：width、height、margin、padding、border、background；
  3. 文本属性：color、font、text-decoration、text-align、vertical-align、white-space、break-word；
  4. 其他属性/CSS3特性：content、cursor、border-radius、box-shadow、text-shadow；
