Nodejs是什么

Node.js是一个基于Chrome V8引擎的JavaScript运行时环境。

Node.js使用事件驱动、非阻塞式I/O模型，轻量高效。

Node.js具备世界上最大开源库生态系统：npm

缺点：单进程，单线程，只支持单核CPU，不能充分利用多核服务器CPU性能；

浏览器端的JS，受制于浏览器提供的接口。比如浏览器提供一个弹对话框的Api，那么JS就能弹出对话框。浏览器为了安全考虑，对文件操作，网络操作，操作系统交互等功能有严格的限制，所以在浏览器端的JS功能是受限于浏览器功能的；

NodeJs完全没有了浏览器端的限制，让Js拥有了文件操作，网络操作，进程操作等功能，和Java，Python，Php等语言已经没有什么区别了。而且由于底层使用性能超高的V8引擎来解析执行，和天然的异步IO机制，让我们编写高性能的Web服务器变得轻而易举。

Node.js环境安装

登陆官网下载对应平台安装包，解压并将安装目录添加至环境变量；

wget http://https://nodejs.org/dist/v16.13.1/node-v16.13.1-linux-x64.tar.xz

xz -d node-v16.13.1-linux-x64.tar.xz
tar xvf node-v16.13.1-linux-x64.tar

export NODE_HOME=./node-v16.13.1-linux-x64
export PATH=$NODE_HOME/bin:$PATH

# 验证
node --version

Node.js核心模块

核心模块即为封装在node.js中的相关模块

fs

fs是文件操作相关的模块

var fs = require('fs')

/*
第一个参数为读取文件路径
第二个参数为回调函数，回调函数接收两个形参：
    err：错误对象，读取成功是为null，失败则为失败信息对象
    data：数据对象，读取成功则为数据内容对象，失败则为null
          读取内容为十六进制内容，可通过toString方法转换为人类可正常识别内容
*/
fs.readFile('../package.json',function (err,data){
    console.log(data.toString())
})

/*
第一个参数：数据写出路径
第二个参数：写出内容
第三个参数：回调函数（仅接受一个错误对象形参）
 */
fs.writeFile('./fswrite.txt','hello,node.js!',function (err) {
    console.log(err)
})

模块规范

模块加载

  • 语法：const 变量名 = require('模块名称或文件名')
    • 执行加载模块中的代码
    • 得到被加载模块中指定导出的对象
  1. node中没有全局作用域，仅有模块作用域；
  2. require优先从缓存加载，提升性能；
  3. 引用核心模块仅需提供模块名即可；
  4. 引用第三方模块类似核心模块，但第三方模块需提前安装；
  5. 引用自定义模块，需提供路径及模块文件名，文件名后缀可省略；
  6. 加载规则为：
    1. 同级目录node_modules/模块名文件夹/package.json中main属性指定文件
    2. 同级目录node_modules/模块名文件夹下index.js
    3. 上级目录node_modules/模块名文件夹/package.json中main属性指定文件
    4. 上级目录node_modules/模块名文件夹下index.js
    5. 直至根目录仍未完成加载则报错

模块导出

  • 语法一（导出对象，可包含多个成员）：exports.foo = ‘hello’
  • 语法二（导出单个成员）：module.exports = ‘hello’
    • 模块中对象需通过挂载至exports对象上，方可支持跨模块访问；
    • exports实际上是module.exports的一个引用；
var ex = require('./exports')
console.log(ex.foo)
// exports.js
var foo = 'hello'
exports.foo = 'node.js'

Node.js包管理器

npm是node package manager的缩写，用来管理node第三方包；通过npm安装第三方包后，会自动生成以下对象：

  • node_modules：存放第三方包
  • package.json：
    • 记录项目初始化相关信息，如项目名称、版本号、描述等；
    • 记录第三方包相关内容，如包名、版本号、依赖项等；
    • 作为npm i批量安装列表

常用命令

  • npm init：生成package.json并完成项目初始化；
  • npm install：安装package.json中依赖第三方包；
  • npm install 包名：仅下载第三方包；
  • npm install 包名 --save：下载第三方包并添加至package.json中；
  • npm uninstall 包名：删除指定第三方包，但保留依赖项；
  • npm uninstall 包名 --save：删除指定第三方包并删除依赖项；

npm install可简写为npm i

npm install 包名 --save可简写为 npm install 包名 -S

npm uninstall 包名可简写为npm un 包名
